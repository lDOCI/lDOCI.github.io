# Слайсер и профили

## Какой слайсер выбрать

| Слайсер | Особенности |
|---|---|
| **[OrcaSlicer](https://github.com/SoftFever/OrcaSlicer/releases)** | Самый современный, регулярно обновляется, профиль AD5M/Pro встроен. Новичку — он. |
| **[Orca-Flashforge](https://wiki.flashforge.com/en/Orca-Flashforge-and-Flashmaker/orca-flashforge-quick-start-guide)** | Орка, модифицированная FF: встроенная веб-морда (аля-Bambu), отправка через облако, работа в паре с приложением **FlashMaker**. |
| **[PrusaSlicer](https://github.com/prusa3d/PrusaSlicer/releases)** | Классика, если привыкли к нему. |

> **Примечание:**
> Раньше считалось, что обычный OrcaSlicer плохо отправляет печать на FlashForge по Wi-Fi. На деле всё работает, если включить **проверку контрольной суммы MD5** (см. [Исправление ошибок → битые файлы](Исправление-ошибок.md#битые-файлы-печати--проверка-md5)) или заходить на принтер по **SSH** ([как получить root/SSH](Прошивка.md#root-доступ-ssh)). Альтернативы — Orca-Flashforge (через облако/приложение) или печать с флешки.

**Подключение Orca-FF и FlashMaker:** в приложении/слайсере логинимся, сканируем QR на принтере для привязки. Для Orca-FF есть русификация (ищется в сообществе).

> **Внимание:**
> В профиле **Orca-FF для AD5X** ретракт по умолчанию стоит **2 мм** — на директ-подаче это много и вызывает микропробки при множестве ретрактов (мелкие детали). Снижайте до **0.6 мм** и меньше.

## Стартовый G-code

Штатный старт можно улучшить, чтобы не было «соплей» и был чистый первый слой:
- Сопло при опускании одновременно едет в угол стола — «кладёт» вытекший пластик на стол, а не на модель (см. [Печать](Печать.md)).
- Очистка сопла (`clean_nozzle`/intro-линия) перед печатью набирает давление в хотенде — мелкие детали стартуют без артефактов.

Многие пишут собственный `PRINT_START`, который греет стол/сопло, грузит карту стола по температуре и чистит сопло:
```
; в Orca → Start G-code:
PRINT_START B=[first_layer_bed_temperature] E=[first_layer_temperature]
```
с макросом, который по температуре стола выбирает нужный профиль карты (`BED_MESH_PROFILE LOAD=...`) и зовёт очистку сопла.

## Оставшееся время на дисплее

Если время печати не показывается, добавьте M73:
```
; Start G-code:
M73 P0 R{total_time_in_minutes}
; After Layer Change G-code:
M73 P{progress_percentage} R{remaining_time_in_minutes}
```
Если прошивка не поддерживает M73 — текстом через `M117 Time left: {remaining_time_in_minutes}m` (индикатор не обновит, но текст покажет).

## Исключение объектов (Exclude Objects)

Удобно «отменять» провалившиеся детали на лету. Но:

> **Внимание:**
> На урезанном Klipper исключение объектов может **повесить принтер после возобновления** печати с паузы. Если печатаете со сменой цвета/паузами — лучше отключить. В названиях объектов на столе не используйте кириллицу.

## Fluidd: голова бьётся о края при ручном перемещении

*(Джог / jog — это ручное перемещение головы и стола кнопками-стрелками в веб-интерфейсе Fluidd/Mainsail.)*

В Fluidd кнопки перемещения могут увести голову/стол за пределы поля и стукнуть в концевики. Лечится переносом макроса `_CLIENT_LINEAR_MOVE` из `client.cfg` в `base.cfg` с правкой реальных пределов:
```
[gcode_macro _CLIENT_LINEAR_MOVE]
description: Linear move with save and restore of the gcode state
gcode:
 {% set e_move = "E" ~ params.E if params.E is defined else "" %}
 {% set rate = "F" ~ params.F if params.F is defined else "" %}
 {% set ABSOLUTE = params.ABSOLUTE | default(0) | int != 0 %}
 {% set ABSOLUTE_E = params.ABSOLUTE_E | default(0) | int != 0 %}
 SAVE_GCODE_STATE NAME=_client_movement
 {% if x_move or y_move or z_move %}
 G90
 {% endif %}
 {% if e_move %}
 M8{ 2 if ABSOLUTE_E else 3 }
 {% endif %}
 # реальные пределы (а не парковочные):
 {% set min = {'x': -110, 'y': -110, 'z': 0} %}
 {% set max = {'x': 110, 'y': 110, 'z': 220} %}
...
 G1 { x_move } { y_move } { z_move } { e_move } { rate }
 RESTORE_GCODE_STATE NAME=_client_movement
```

## Веб-камера в OrcaSlicer на стоке

Картинку с USB-камеры можно вывести в OrcaSlicer и на стоковой прошивке (через настройку URL потока). Список рабочих камер — в [Прошивке](Прошивка.md#камеры-usb).
